# CHZZK-Chat-Monitor
방송인 혹은 유튜버 혹은 키리누커를 위한 프로그램 일지도 모르는 것 Ver.1.0.0
치지직 라이브 방송의 채팅을 실시간으로 수집하고, 설정한 키워드의 발생과 방송 진행 시간을 함께 기록해 주는 모니터링 도구입니다.
예를 들어 e스포츠 중계팀에서 “펜타킬” 같은 특정 키워드가 나오면 즉시 하이라이트 후보로 표시하고 싶을 때 이 프로그램을 실행합니다. 방송 주소에서 채널 ID를 입력하고, 브라우저에서 복사한 `NID_AUT`, `NID_SES` 쿠키 값을 채워 넣은 뒤 키워드 설정란에 `펜타킬:2:120, 역전승:1:300`처럼 입력하고 `시작` 버튼을 누릅니다. 그러면 방송 중 채팅이나 후원 메시지에서 해당 키워드가 2분 안에 2번 이상 감지될 때마다 `keyword_times.log`에 기록되고, 동시에 진행 시간이 `01:23:45`처럼 함께 남아 나중에 편집자가 손쉽게 타임라인을 추적할 수 있습니다.

키워드를 입력은 키워드:횟수:시간 이렇게 셋으로 구분 되어 있으며 : <- 세미콜론 으로 구분합니다.

키워드는 방송 채팅에서 탐지하고 싶은 단어나 문장을 말합니다. 예를 들어 “펜타킬”이나 “우승”처럼 특정 순간을 기념하는 단어를 지정할 수 있습니다. 횟수(임계치)는 해당 키워드가 주어진 시간 동안 몇 번 이상 등장해야 기록할지를 정하는 값입니다. 예를 들어 횟수를 3으로 설정하면 같은 키워드가 설정된 시간 동안 최소 3번은 등장해야 로그에 남습니다. 시간(윈도)은 키워드가 몇 초 동안 탐지되어야 같은 카운트로 인정할지를 의미합니다. 시간 120초와 횟수 3을 함께 설정하면, 2분 동안 키워드가 3번 이상 등장했을 때만 이벤트로 기록합니다. 이렇게 키워드, 횟수, 시간을 조합하면 “어떤 상황을 어느 정도 빈도로 포착할 것인지”를 세밀하게 제어할 수 있습니다.

키워드에는 기본 키워드와 포함 키워드가 있습니다.
기본 키워드는 단어 경계를 기준으로 정확히 일치할 때만 감지되는 키워드입니다. 예를 들어 기본 키워드에 `우승`을 넣어두면 “우승 했습니다” 같은 문장에서만 카운트되고, “준우승”처럼 단어 일부에만 포함된 경우는 제외됩니다. 덕분에 명확한 조건으로 특정 표현을 추적할 수 있습니다.

포함 키워드는 단어 경계를 무시하고 문자열 안에 해당 단어가 들어가기만 하면 감지되는 키워드입니다. 예를 들어 포함 키워드에 `승`을 넣어두면 “우승”, “준우승”, “승리”처럼 어떤 단어에 포함돼도 카운트됩니다. 덕분에 철자 변형이나 붙임표, 이모티콘 등이 섞여도 폭넓게 탐지할 수 있습니다. 이런 두 방식을 상황에 맞게 병행하면, 정확도가 필요한 키워드와 대략적인 분위기를 읽고 싶은 키워드를 각각 유연하게 설정할 수 있습니다.

프로그램을 실행하면 아래와 같이 파일들이 생기게 됩니다.

<img width="213" height="139" alt="image" src="https://github.com/user-attachments/assets/9bfc8e53-27c9-45ce-b192-e822e1597b16" />
프로그램을 실행하면 다음과 같은 파일들이 만들어지거나 갱신됩니다.

- `keyword_times.log`
설정한 키워드가 임계치를 충족했을 때의 이벤트가 한 줄씩 기록됩니다. 각 줄에는 발생 시각, 채널명, 키워드, 닉네임, 방송 진행 시간, 메시지가 탭으로 구분돼 저장되어, 나중에 타임라인을 재구성하는 데 유용합니다.

- `broadcast_times.log` (옵션)
`broadcast_logger.py`를 직접 실행했을 때 생성됩니다. 방송 시작 시각과 현재 시각, 방송 진행 시간이 기록되어 방송 상태를 모니터링할 수 있습니다.

- `cookies.json`
GUI에서 입력한 `NID_AUT`, `NID_SES` 값이 저장됩니다. 다음 실행 시 자동으로 불러와 편리하게 사용할 수 있지만, 보안상 외부에 공유되면 안 됩니다.

- `keyword_settings.json`
GUI에서 설정한 키워드/임계치/시간 정보가 저장됩니다. 프로그램 재실행 시 이전 설정을 불러오기 때문에 매번 다시 입력할 필요가 없습니다.

이처럼 로그·설정·빌드 결과 등이 자동으로 관리되므로, 프로그램을 중단했다 재실행해도 이전 상태를 이어받아 사용할 수 있습니다.

채팅이 keyword_times.log에 저장되면 아래와 같은 양식으로 저장 됩니다.
순서대로 방송 날짜, 채널 명, 키워드, 채팅친사람 닉네임, 스트리밍 시간, 올라온 채팅 이렇게 구성되어 있습니다.

프로그램 안에도 설명서가 구비 되어 있으니 그걸 보셔도 사용하는데 문제는 없을 겁니다.(아마도)

프로그램 파일을 여러번 실행시키면 여러개의 방송을 동시에 기록 할 수 있습니다. 하지만 그렇게 되면 keyword_times.log의 내용이 상당히 복잡해짐으로 추천 하는 방법은 폴더를 여러개 만들고 프로그램을 각각의 폴더 안에 넣어서 프로그램을 작동시키는 방법입니다. 물론 하나의 폴더에서 프로그램을 여러번 실행시켜도 문제 없이 여러개의 창이 나타나며 하나의 keyword_times.log에 제대로 기록이 됩니다.

방송이 꺼진 상황에서도 이 프로그램을 켜두어도 아무 문제가 없습니다. 방송이 꺼졌다 다시 켜진다고 하여도 문제 없이 작동합니다. 하지만 가끔 NID_AUT와 NID_SES가 바뀌는 경우가 있으니 작동이 제대로 되지 않는다면 확인해주시길 바랍니다.
